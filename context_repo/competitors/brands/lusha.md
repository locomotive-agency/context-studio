## Summary

Lusha is a B2B contact-data + buying-signals platform for GTM teams, structured as three product surfaces — **Workspace** (search/enrich/manage), **API** (programmatic), and **Extension** (LinkedIn + browser). The most strategically interesting Lusha development is **Lusha MCP** — Lusha ships a Model Context Protocol server, putting it in the same emerging MCP-for-sales lane as ZoomInfo and Clay (only three competitors in the broader Tier 1 set with documented MCP offerings). Pricing is fully public with five tiers (Free, Starter $37.45, Pro $52.45, Premium $299.95, Scale custom), serving 300,000+ GTM teams. **Heavy Cloudflare blocking on most product pages prevented full extraction** — this dossier is partial; pricing, MCP positioning, and homepage framing are reliable, but product-detail depth on Workspace/API/Extension is gappy.

## Product Offerings

### [[lusha--workspace|Lusha Workspace]]

- **What it does:** Centralized surface for searching, enriching, and managing prospects — accounts and contacts in one place.
- **Key features:**
  - Search across Lusha contact + company data
  - Enrichment workflows
  - Contact + account management
  - Pre-built workflows for common GTM actions
  - Find, enrich, and manage prospects in one canvas
- **Source URL:** https://www.lusha.com/

### [[lusha--api|Lusha API]]

- **What it does:** Programmatic access to Lusha data — runs workflows and plays with accurate data, plugged into customer systems.
- **Key features:**
  - Programmatic data access
  - Workflow + play execution
  - Built for embedding into internal tools / CRM
  - Available as a separate product surface
- **Source URL:** https://www.lusha.com/lusha-api/ (referenced)

### [[lusha--extension|Lusha Extension (Chrome Extension)]]

- **What it does:** Browser extension that reveals contact details on LinkedIn and other sites — Lusha's flagship UX surface.
- **Key features:**
  - LinkedIn + Sales Navigator overlay
  - Reveal contact details while browsing
  - One-click capture
  - Most-used Lusha surface historically
- **Source URL:** https://www.lusha.com/lusha-extension/ (extraction blocked — see flags)

### [[lusha--mcp|Lusha MCP Server]]

- **What it does:** Model Context Protocol server that exposes Lusha data to AI assistants like Claude — enables natural-language sales workflows powered by Lusha contact + buying-signal data.
- **Key features:**
  - Native MCP integration
  - AI assistant access (Claude, ChatGPT, others)
  - Sales-specific use cases (closed-won lookalike, account research)
  - Recently launched joint Clay × Lusha livestream demonstrating combined Clay + Lusha + MCP workflows
- **Source URL:** https://www.lusha.com/blog/lusha-mcp-sales-use-cases/

### Buying Signals

- **What it does:** Track real-world changes — job changes, funding, hiring, technology shifts — to surface buying-trigger moments.
- **Key features:**
  - Job change tracking
  - Funding / M&A signals
  - Hiring signals
  - Technographic shifts
- **Source URL:** https://www.lusha.com/buying-signals/ (referenced)

## Differentiators (vs the broader category)

- Lusha MCP server — one of only three Tier 1 ZoomInfo competitors (alongside ZoomInfo itself and Clay) with a documented MCP offering. Strategic future-positioning play (source: https://www.lusha.com/blog/lusha-mcp-sales-use-cases/).
- Public, fully transparent pricing with $ amounts at four of five tiers — most transparent commercial model among contact-data vendors (source: https://www.lusha.com/pricing/).
- 300,000+ GTM teams as customer base — strong SMB/individual penetration (source: https://www.lusha.com/).
- Strong Chrome extension UX legacy — Lusha was a pioneer in browser-extension-based contact reveal.
- Built-in compliance: GDPR, CCPA, ISO 31700, SOC 2 Type 2 (source: https://www.lusha.com/).
- Three-product surface (Workspace + API + Extension) — clean architecture vs single-surface competitors.
- Active partnership with Clay — joint Clay × Lusha livestreams demonstrating combined orchestration patterns.

## Crossover With ZoomInfo

- **Direct overlap:**
  - **ZoomInfo Data foundation (Pillar 1) ↔ Lusha Workspace + API.** Direct head-to-head on B2B contact data — Lusha's Workspace and API surfaces target the same use cases as ZoomInfo's data + Operations products. (source: https://www.lusha.com/)
  - **ZoomInfo APIs & MCP ↔ Lusha API + Lusha MCP.** Both ship MCP servers — strong direct overlap in the agent-data lane. Lusha is one of very few competitors with a strategic MCP commitment. (source: https://www.lusha.com/blog/lusha-mcp-sales-use-cases/)
  - **ZoomInfo Sales / GTM Workspace seller surface ↔ Lusha Extension + Workspace.** Both deliver in-workflow prospecting from LinkedIn into CRM. Lusha's extension legacy is stronger in browser UX; ZoomInfo's strength is the data depth.
  - **ZoomInfo Operations / GTM Studio (data ops) ↔ Lusha API + Workspace enrichment.** Both target programmatic and bulk enrichment workflows.
  - **ZoomInfo Intent / Job Change / Buying Signals ↔ Lusha Buying Signals.** Both surface real-world buying-trigger events. (source: https://www.lusha.com/buying-signals/)
- **Adjacent overlap:**
  - **ZoomInfo's "team of AI agents" framing ↔ Lusha + Clay + MCP integration patterns.** Lusha's ecosystem play is to be the data layer inside agent workflows orchestrated by Clay; ZoomInfo wants to be the data layer + agent layer inside its own platform.
- **No overlap:**
  - **ZoomInfo Chorus** — no CI product.
  - **ZoomInfo Marketing FormComplete / WebSights / ABM platform** — no marketing-side product.
  - **Conversation intelligence** — not in scope.
- **Their pitch against ZoomInfo (if found):** _No public Lusha-authored vs-ZoomInfo page surfaced in the extracted pages._ Lusha publishes a `/alternatives/` competitive content surface (referenced but not extracted) which likely includes ZoomInfo positioning. Competitive framing is implicit through pricing transparency and MCP-forward positioning.

## Messaging & Positioning Notes

- **Recurring proof points:** 300,000+ GTM teams; SOC 2 Type 2 + ISO 31700 + GDPR + CCPA compliance; partnership with Clay (closed-won lookalike engine livestream).
