# Context workbench value proposition

The context workbench helps enterprise teams give AI workflows the right context without losing control of approved truth.

## Core value

Teams can keep approved marketing context in OKF Documents while using Collections to search supporting material like sales conversations, research notes, and competitive files.

## What changes for users

- Marketers stop rebuilding prompt context from scattered documents.
- Sales enablement can use customer language from transcripts without treating it as approved proof.
- Brand and legal teams can see which controlled records are used.
- AI workflow owners can test the actual context package response before production use.
- Viewers can request context and semantic data without gaining edit access.

## Differentiators

- Deterministic retrieval for approved context.
- Semantic retrieval only where it belongs: supporting Collections.
- Cited passages from source documents.
- Plain role model for Admins, Editors, and Viewers.
- Local-first prototype that can run with Git-backed OKF records and SQLite state.

## Short pitch

Give every AI workflow a clean context package: approved Documents for trusted guidance, cited Collections for supporting evidence, and a test bench to inspect what the workflow will receive.
