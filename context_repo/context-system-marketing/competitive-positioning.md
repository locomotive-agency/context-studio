# Context system competitive positioning

## Compared with a generic knowledge base

Generic knowledge bases are good for browsing and documentation. They usually do not separate approved context from raw source material in a way AI workflows can enforce.

Position the system as a governed context workbench, not a general documentation site.

## Compared with vector search over shared drives

Vector search is useful for finding related passages, but it is a weak control plane for approved messaging. It can retrieve stale, draft, or conflicting source material if all files are treated the same.

Position Collections as the place where similarity search belongs, and Documents as the place where approved context lives.

## Compared with prompt libraries

Prompt libraries help users reuse instructions, but they do not solve context freshness, scope, status, or source citation.

Position the system as context infrastructure for prompts and AI workflows.

## Compared with a full enterprise CMS

Enterprise CMS tools manage publication workflows and web content. This system manages AI-facing context packages and supporting retrieval.

Position it as complementary infrastructure for AI operations, marketing operations, and enablement teams.

## Competitive traps

- Do not get pulled into feature-by-feature CMS comparisons.
- Do not claim the system is a complete vector platform.
- Do not promise that all governance happens automatically.
- Do emphasize inspectable context packages and clear data boundaries.
