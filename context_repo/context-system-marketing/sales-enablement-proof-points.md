# Context system enablement proof points

These are synthetic proof points for demos and tests. They are not customer claims.

## Approved demo proof points

- A marketing operations team can create approved Documents for messaging, proof points, and legal guardrails.
- An enablement team can create a Collection of sales transcripts and search it for cited buyer language.
- A viewer can request context and semantic data without editing the knowledgebase.
- A Tool Test Bench user can inspect the same response an AI workflow receives.
- Controlled context can block a run when required approved guidance is missing.

## Example demo scenarios

Use these when showing the system:

- Search sales conversations for "approved truth versus raw transcript notes."
- Ask for context for a campaign brief targeting enterprise AI operations leaders.
- Show how a Collection passage is cited separately from OKF records.
- Show that a Viewer can request context but cannot save a Document.

## Claim guardrails

- Say "synthetic demo proof point" when using this content externally.
- Do not present the fake companies or transcripts as real customer evidence.
- Do not imply real adoption, revenue, or performance metrics.
