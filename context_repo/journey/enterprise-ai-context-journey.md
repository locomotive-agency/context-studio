# Enterprise AI Context Buying Journey

## Stages

1. AI experimentation creates useful but inconsistent outputs.
2. Teams identify that context quality, not model quality, is the main blocker.
3. Governance owners separate approved guidance from raw source material.
4. Editors build a small Document set and connect supporting Collections.
5. Viewers test context retrieval before workflows are deployed.

## Key Questions

- Which records are approved?
- Which sources are raw evidence?
- Can the agent retrieve full text only when it needs it?
- Can non-admin editors maintain the repository?
